# Лабораторная работа 1: *Создание и развертывание статического сайта на базе MkDocs с публикацией на GitHub Pages*
## Цели
1. Освоить процесс создания статического сайта с использованием генератора документации **MkDocs**.
2. Научиться организовывать структуру документации проекта (портфолио лабораторных работ).
3. Закрепить базовые принципы работы с системой контроля версий **Git** и платформой **GitHub**.
4. Развернуть статический сайт с использованием механизма **GitHub Pages** на домене вида `username.github.io`.
5. Освоить базовую настройку темы оформления и конфигурационного файла `mkdocs.yml`.

## Задачи
В рамках лабораторной работы требовалось:

- Создать публичный репозиторий GitHub для размещения сайта и настроить публикацию сайта через **GitHub Pages** из каталога `/docs` ветки `main`
- Клонировать репозиторий на локальный компьютер, создать и активировать виртуальное окружение Python, установить **MkDocs**
- Настроить файл `.gitignore`
- Создать новый проект командой `mkdocs new source` и выполнить локальный запуск сайта с помощью `mkdocs serve`
- Выбрать и подключить тему оформления, настроить файл `mkdocs.yml`, реализовать логичную структуру сайта и меню, а также создать отдельные страницы для лабораторных работ
- Собрать сайт в каталог `/docs` командой `mkdocs build -d ../docs`
- Выполнить публикацию проекта в GitHub

## Ход работы
### 1. Подготовка проекта и виртуального окружения

На первом этапе был создан локальный репозиторий проекта, после чего в корневой директории было настроено виртуальное окружение Python. Это позволило изолировать зависимости проекта и установить **MkDocs** отдельно от глобального окружения системы.

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install mkdocs
```

### 2. Создание проекта MkDocs

После установки MkDocs был создан новый сайт в отдельной папке `source`.

```bash
mkdocs new source
cd source
mkdocs serve
```

Такой подход позволил разделить исходные файлы сайта и итоговую сборку для **GitHub Pages**. В результате была сформирована начальная структура проекта с файлом конфигурации `mkdocs.yml` и каталогом `docs/`, содержащим Markdown-страницы:

```bash
source/
├── mkdocs.yml
└── docs/
    └── index.md
```

### 3. Настройка файла mkdocs.yml

Следующим шагом была настройка файла `mkdocs.yml`.
В нём были заданы:

- название сайта
- тема оформления
- цветовая палитра
- структура меню через nav
- дополнительные возможности темы Material
- расширения Markdown для улучшения оформления страниц

```bash
site_name: Портфолио по лабораторным работам

theme:
  name: material
  language: ru
  palette:
    scheme: slate
    primary: purple
    accent: deep purple
  features:
    - navigation.tabs
    - navigation.sections
    - search.suggest

nav:
  - Главная: index.md
  - Об авторе: about.md
  - Лабораторные работы:
      - Лабораторная работа 1: labs/lab1.md
      - Лабораторная работа 2: labs/lab2.md
      - Лабораторная работа 3: labs/lab3.md

markdown_extensions:
  - admonition
  - pymdownx.details
  - pymdownx.superfences
```

### 4. Организация структуры страниц

В соответствии с требованиями была создана многостраничная структура сайта.
Итоговая структура исходных файлов выглядит следующим образом:

```bash
source/docs/
├── index.md
├── about.md
└── labs/
    ├── lab1.md
    ├── lab2.md
    └── lab3.md
```

На странице каждой лабораторной работы был использован единый шаблон отчёта:

- Цели
- Задачи
- Ход работы
- Выводы

??? info "Пояснение"
    Благодаря использованию параметра nav и продуманной структуры файлов удалось реализовать логичную иерархическую навигацию, соответствующую требованиям ЛР.

### 5. Сборка сайта для GitHub Pages

После завершения настройки страниц и конфигурации была выполнена сборка сайта в каталог `/docs` корня репозитория.

```bash
mkdocs build -d ../docs
```

После выполнения команды в корне репозитория была сформирована готовая версия сайта, содержащая HTML-файлы, стили, скрипты и вспомогательные ресурсы:

```bash
breakthr0ugh.github.io/
├── docs/
│   ├── about/ 
│   ├── assets
│   ├── labs/
│   ├── search
│   └── index.html
└── source/
    ├── mkdocs.yml
    └── docs/
```

### 6. Публикация изменений в GitHub

Завершающим этапом стала фиксация изменений в Git и их отправка в удалённый репозиторий для дальнейшей публикации обновлённой версии сайта.

В процессе работы была настроена структура проекта, организованы страницы сайта, реализовано меню навигации и выполнена публикация итоговой версии сайта.
